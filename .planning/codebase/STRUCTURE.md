## Directory Purposes

**`src/`:**
- Purpose: React application source
- Contains: `App.tsx`, `main.tsx`, `App.css`, hooks, components, and TS types
- Key files: `codex-switcher-origin/src/App.tsx`, `codex-switcher-origin/src/hooks/useAccounts.ts`
- Subdirectories:
  - `components/` for cards and modals
  - `hooks/` for Tauri command orchestration
  - `types/` for frontend-facing shared types

**`src-tauri/src/`:**
- Purpose: Rust backend modules
- Contains: command handlers, auth/storage modules, API clients, scheduler, and shared types
- Key files: `codex-switcher-origin/src-tauri/src/lib.rs`, `codex-switcher-origin/src-tauri/src/types.rs`
- Subdirectories:
  - `commands/` for Tauri-exposed functions
  - `auth/` for storage, switching, OAuth, token refresh, and settings
  - `api/` for external API clients

**`.github/workflows/`:**
- Purpose: Build and release automation
- Contains: `build.yml`

## Naming Conventions

**Files:**
- `PascalCase.tsx` for React components such as `AccountCard.tsx`
- `camelCase.ts` for hooks/modules such as `useAccounts.ts`
- `snake_case.rs` for Rust modules such as `account.rs`, `token_refresh.rs`
- `index.ts` barrel exports for frontend component aggregation

**Directories:**
- Lowercase directory names with simple domain grouping: `components`, `hooks`, `types`, `commands`, `auth`, `api`

**Special Patterns:**
- Frontend type names mirror Rust payload fields and often preserve `snake_case` property names
- Command modules are re-exported through `codex-switcher-origin/src-tauri/src/commands/mod.rs`

## Where to Add New Code

**New Frontend Feature:**
- Primary code: `codex-switcher-origin/src/components/` and `codex-switcher-origin/src/App.tsx`
- Native bridge changes: `codex-switcher-origin/src/hooks/useAccounts.ts`
- Types: `codex-switcher-origin/src/types/index.ts`

**New Native Command:**
- Definition: `codex-switcher-origin/src-tauri/src/commands/*.rs`
- Registration: `codex-switcher-origin/src-tauri/src/lib.rs`
- Domain logic: `codex-switcher-origin/src-tauri/src/auth/`, `api/`, or `scheduler.rs`

**Utilities / Shared Native Logic:**
- Shared helpers: `codex-switcher-origin/src-tauri/src/auth/fs_utils.rs`
- Domain models: `codex-switcher-origin/src-tauri/src/types.rs`
