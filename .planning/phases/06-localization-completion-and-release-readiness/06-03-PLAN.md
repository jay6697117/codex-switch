# 06-03 Plan: macOS Packaging and Release Verification

## Goal

建立首个公开发布所需的 macOS release pipeline：universal build、DMG 产物、签名、公证、校验值、GitHub Releases 草稿以及最终 packaged-app 验证。

## Deliverables

- 为 `wails.json` 和 `build/darwin` 补齐 release 所需的 macOS metadata、bundle identity 和 signing/notarization 相关资产。
- 新增独立的 macOS release workflow，执行 universal build、DMG generation、codesign、notarization、staple/verify、checksum generation 和 release asset upload。
- 把 app version 改成明确的 release source of truth，由 tag 或 build-time env 注入，而不是继续依赖开发态常量。
- 通过 GitHub Releases 草稿交付 `DMG + checksums + install/upgrade notes`，不把正式发布前置到手工命令里。
- 增加 packaged-app manual smoke checklist，验证 signed/notarized 产物在真实 macOS 上的安装、首启和核心 shell 可用性。

## Verification

- `go test ./...`
- `cd frontend && npm run typecheck`
- `cd frontend && npm test`
- `cd frontend && npm run e2e`
- `wails build -clean`
- macOS universal signed/notarized artifact verification in release workflow
