# 06-02 Plan: Parity Regression Matrix

## Goal

把现有 Go、frontend 和 Playwright 自动化从“开发期功能验证”提升为“公开发布前的核心流 parity regression”，覆盖所有已交付能力域。

## Deliverables

- 扩展 Go service regressions，重点覆盖 auth/switch/backup/warmup/settings 中与 Phase 6 发布质量直接相关的边界和错误语义。
- 扩展 frontend integration tests，覆盖双语下的核心渲染、错误映射、success feedback、fallback observability、settings-driven locale change，以及 full backup 成功路径。
- 扩展 Playwright regression，使账号管理、切换、OAuth、usage、warmup、backup、settings 的核心流都进入 release gate，并明确区分“mocked shell smoke”和“需要 packaged runtime 才能验证的最终行为”。
- 形成 requirement-to-test 的清晰映射，确保 `AUTH-05`、`I18N-02`、`I18N-05`、`QUAL-01` 不再停留在隐式覆盖状态。
