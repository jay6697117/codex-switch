# 06-01 Plan: Localized Message Mapping and Copy Closure

## Goal

把现有 shell 与核心 flows 的双语覆盖收口到发布标准，补齐 backend message code 差集，并为账号添加/切换提供明确的 localized success and failure feedback。

## Deliverables

- 审计并补齐 `accounts/auth/backup/settings/shell/usage/warmup/errors` 的核心 UI copy 覆盖，消除主流程中的 mixed-language。
- 补齐当前未进入 `errors` namespace 的 backend error codes，至少包括：
  - `auth.account_not_found`
  - `auth.refresh_failed`
  - `process.stop_failed`
  - `process.restart_failed`
- 把 add-account 和 account-switch flows 的 success/failure feedback 形成稳定闭环，优先使用 `ResultEnvelope.message` 或等价的结构化 message mapping，而不是只依赖隐式状态变化。
- 把 full backup 原生文件对话框标题、usage/warmup 时间格式和 `usage.planType` 这类 raw display field 纳入 app-locale 策略，不再默认跟随系统 formatter 或 provider 原始字符串。
- 建立两道 guardrail：
  - backend `AppError` code 必须在 `errors` namespace 中有映射
  - `zh-CN` / `en-US` 资源 key shape 必须保持一致
- 为缺失翻译建立可观测机制，避免 `zh-CN` 主流程在运行时静默掉到空白或 raw key。
